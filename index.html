Overview

WebXR for WebGL - is a plugin for Unity game engine that enables you to be able to create AR Experiences in Unity and run them on mobile web browsers. You can easily create AR experiences or even AR game that interact realistically with objects and environments.

Key Features

WebXR for WebGL plugin enables you to be able to create next AR Experiences:

  • Targets Recognition - for recognition and tracking of prepared images.
  • Targets Capture - for recognition and tracking targets defined by users in runtime.
  • Face recognition - for recognition the position of the face and facial expressions.
  • Surface recognition - for surface recogntion.
  • AR Space - for navigation through the AR scene based on the movement of the user's device.
  • AR Geolocation - for tracking the direction and distance to geolocations based on the user's location.

Getting Started

To use the WebXR for WebGL plugin you will need to install the plugin. Visit releases section on Github (https://github.com/Qualium-Systems-Ltd/WebXR-for-WebGL/releases/) and download the latest plugin release. And you will need to create an empty Unity project.

Note

The WebXR for WebGL plugin has been developed with Unity 2021.3.13f1 using the URP (Universal Render Pipeline), but URP is not neccessary. In addition, some features of URP (such as Post-Processing and HDR) is not supported in the plugin.

Installation

  • Open created empty project.
  • Press 'Assets' menu tab and from drop-down list, select option 'Import Package', from next drop-down, select option 'Custom Package...'
  • Select downloaded package and press 'Open'.
  • In the left bottom corner of opened window press 'All' and after that press 'Import' button.
  • Wait for the import process to finish.
  • Done.

Next Steps

Now you can select wich feature you want to implement.

  • Targets Recognition - for recognition and tracking of prepared images.
  • Targets Capture - for recognition and tracking targets defined by users in runtime.
  • Face recognition - for recognition the position of the face and facial expressions.
  • Surface recognition - for surface recogntion.
  • AR Space - for navigation through the AR scene based on the movement of the user's device.
  • AR Geolocation - for tracking the direction and distance to geolocations based on the user's location.

Or if you have any question check FAQs section.

Scene Setup

  • In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Targets Tracking' option.
  • 'TargetsController' game object has been created. As you can see there are some errors in console, but at this stage ignore them.
  • In Hierarchy press 'Righ mouse button (RMB)', from drop-down list select 'AR-Renderer', from next drop-down list select 'Misc', from next drop-down list select 'Target' option.
  • Set created target as child of 'AR-Renderer-Scene' game object. After that select 'TargetController' game object and in Inspector you can see 'Target Track Controller (Script)' attached to it.
  • Drag and drop created target into 'Targets' field of 'Target Track Controller (Script)' component in Inspector. So now 'Targets' field contains two targets.
  • Navigate to 'Assets/Compiler/Targets' folder and put your target image into this folder.
  • Now expand created target in Hierarchy and select 'AR-Renderer-Target-Preview' game object. In Inspector you can see 'Target Preview (Script)' attached to it.
  • Drag and drop your target image into 'Image' field of 'Target Preview (Script)' component. Your target image should appear in Scene view. You can use your target image in Scene view as real-world size reference.
  • For now you can disable default target in Hierarchy.
  • In Hierarchy expand created target and select 'Displayer' game object. Set whatever you need as child of this game object. In my case it will be a sphere.
  • Enable default target in Hierarchy if you disabled it earlier.
  • Save your scene. After that open 'File' menu tab and from drop-down list select 'Build Settings...' and add your scene to build.
  • Now when scene is ready we can proceed to Targets Setup section.

Targets Setup

  • In Project overview window navigate to 'Assets/Compiler' and press 'Righ mouse button (RMB)', from drop-down list select 'Show in Explorer' option.
  • In explorer window navigate to '.../Assets/Compiler' folder/ Here you can find 'DotMindCompiler-win' for Windows, 'DotMindCompiler-linux' for Linux and 'DotMindCompiler-macos' for macOS. Run the file depending on your system. In my case it will be 'DotMindCompiler-win'
  • In explorer window navigate to '.../Assets/Compiler' folder/ Here you can find 'DotMindCompiler-win' for Windows, 'DotMindCompiler-linux' for Linux and 'DotMindCompiler-macos' for macOS. Run the file depending on your system. In my case it will be 'DotMindCompiler-win'
  • The compiler page will automatically open in your browser. If this does not happen, go to the address from the console window.
  • As you can see this page is empty. Now we will fix it. Go back to Unity and in Hierarchy select 'TargetController'. In Inspector press 'Generate' button of 'Target Track Controller (Script)' component. If everything is correct the log message will be appear in console. New scheme has been created and saved to 'Assets/Compiler/Schemas' folder
  • Now go back to your browser and press 'Refresh data' button. Now you can see all targets on all of your scenes.
  • On this page you can check if everything is okay and if so, press 'Compile .mind file' button. TThe process will start and you will see the progress under the button.
  • When the comilation is finished, you will see 'Done!' message. Instructions (targets_<<scene index>>.mind) file has been created and saved into your 'Assets/Targets' folder.
  • Compiler's page and console window can be closed.
  • Now when targets are ready we can proceed to Settings section.

Settings

  • Open 'File' menu tab and from drop-down list select 'Build Settings...'
  • Now switch build platform to WebGL
  • Go to 'Player Settings...' and navigate to 'Resolution and Presentation' tab. Select 'AR-Renderer' WebGL Template.
  • If neccessary you can set 'Base href' property. (The base URL to be used throughout the document for relative URLs. Absolute and relative URLs are allowed). For now leave it empty.
  • Navigate to 'Other Settings' tab.
  • Uncheck 'Auto Graphic API'
  • Set 'Lightmap Encoding' to 'Normal Quality'
  • Navigate to 'Publishing Settings' tab.
  • Set 'Compression Format' to 'Disabled'
  • Now we can proceed to Build section.

Build

Everything is ready. It's time to check how it works.

  • Open 'File' menu tab and from drop-down list select 'Build Settings...', in opened window press 'Build' button. Select folder and save your build.
  • To run build locally you will need local webserver with https protocol. You can download it here.
  • Download and unpack everything fro m 'local_debug' folder into your build folder.
  • Go to you build folder and run the file depending on your system ('app-windows' for Windows, 'app-linux' for Linux or 'app-macos' for macOS)
  • Open address from the opened console window in browser on your phone.

    • Note

    At this moment only the Chrome browser on Android devices is fully supported.
